Browser WASM Integer Overflow Demo

This demo simulates a data-submission backend implemented in WASM (C). The backend allocates a fixed 32-byte buffer and validates incoming items with the check uint8_t total = item_count × 8. When item_count = 32 the product is 256, which wraps to 0 in uint8_t. The bounds check passes, but memcpy uses the actual size (256 bytes), overflowing the buffer into the adjacent is_admin flag.

Item count (uint8_t, 0–255)

Try 3 for normal behavior, or 32 to trigger the overflow.

Expected behavior

  • item_count = 3: total = 24, check 24 > 32 → reject? No — passes, safe write of 24 bytes.
  • item_count = 5: total = 40, check 40 > 32 → rejected correctly.
  • item_count = 32: total wraps to 0, check 0 > 32 → passes, 256 bytes written → is_admin overwritten → access granted.
