WASM Use-After-Free — AddressSanitizer Build

Countermeasure demo. This page uses the identical vuln.c as the unprotected build. The only difference is the compiler flag -fsanitize=address. ASan instruments every heap operation at compile time and aborts execution the moment a freed memory region is accessed.

What to observe

  1. After Create Session a normal token read succeeds — ASan is silent.
  2. After Logout / Free Session the UAF read is attempted.
  3. ASan detects access to freed memory, writes a heap-use-after-free report to stderr (shown in the log above), and calls abort().
  4. The token is never returned — the module halts before the attacker-controlled data can be read.
